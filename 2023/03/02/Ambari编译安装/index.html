Apache Ambari是一个用于配置、管理和监视ApacheHadoop集群的工具。Ambari由一组RESTful API和基于浏览器的管理界面组成。
Ambari使系统管理员能够：

  1. 配置Hadoop群集
  2. Ambari提供了一个易于使用的分步向导，用于在任意数量的主机上安装Hadoop服务。
  3. Ambari负责集群Hadoop服务的配置。
  4. 管理Hadoop群集
  5. Ambari提供了在整个集群中启动、停止和重新配置Hadoop服务的中央管理。
  6. 监视Hadoop群集
  7. Ambari提供了一个用于监控Hadoop集群运行状况和状态的仪表板。
  8. Ambari将在您需要注意时发送电子邮件（例如，节点故障、剩余磁盘空间不足等）。

Ambari编译

  • 下载源码
    wget https://www-eu.apache.org/dist/ambari/ambari-2.7.7/apache-ambari-2.7.7-src.tar.gz (use the suggested mirror from above)
tar xfvz apache-ambari-2.7.7-src.tar.gz
cd apache-ambari-2.7.7-src
mvn versions:set -DnewVersion=2.7.7.0.0
 
pushd ambari-metrics
mvn versions:set -DnewVersion=2.7.7.0.0
popd

Ambari agent安装

在集群中的每台主机上手动安装并启动 Ambari 代理
安装 Ambari 代理 RPM，在 RHEL/CentOS 上：
yum install ambari-agent/target/rpm/ambari-agent/RPMS/x86_64/ambari-agent-*.rpm
通过编辑主机名行，在 /etc/ambari-agent/conf/ambari-agent.ini 中编辑 Ambari 服务器的位置。
启动 Ambari 代理：ambari-agent start
查看 Ambari 代理日志：tail -f /var/log/ambari-agent/ambari-agent.log

安装Apache Hadoop等组件

网上的Ambari文档大部分都是直接安装HDP版本的软件，但是，HDP目前不开源，所以公司需要使用完全开源的大数据组件
要想使用Amabri安装Apache版本的相应组件，安装步骤应该如下：

  1. 使用Ambari安装HDP版本的rpm包（为了让它创建集群环境，一些service目录等）
  2. 替换已经安装的组件，替换成tar.gz，这种方式的好处是，各个组件解耦，并且不需要编译
  3. 测试替换后组件的稳定性
